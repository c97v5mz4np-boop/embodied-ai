# GitHub 项目发布检查清单

## 仓库与文档

- [ ] README 清楚说明目标、环境、运行方法和当前状态。
- [ ] 目录结构与文档链接有效。
- [ ] 明确区分已完成内容与后续计划。
- [ ] 不包含未经验证的实验结论。

## 可复现性

- [ ] 记录 Python、依赖、系统和硬件版本。
- [ ] 提供最小运行、训练和评估命令。
- [ ] 保存配置、随机种子和评估协议。
- [ ] 检查脚本可从仓库根目录运行。

## 数据、模型与安全

- [ ] 确认数据集、模型权重和第三方代码的许可。
- [ ] 大文件不直接误提交到 Git。
- [ ] 不包含密钥、令牌、个人路径或敏感日志。
- [ ] 对外发布前检查 Git 历史中的敏感信息。

## 展示材料

- [ ] 图表来自真实日志并标注实验设置。
- [ ] 视频与 checkpoint 对应关系清楚。
- [ ] 简历 bullet 与仓库证据一致。
- [ ] 面试讲稿能够解释技术选择和失败案例。
